python使用file模块

在磁盘上读写文件的功能都是由操作系统提供的，现代操作系统不允许普通的程序直接操作磁盘，所以，读写文件就是请求操作系统打开一个文件对象（通常称为文件描述符），然后，通过操作系统提供的接口从这个文件对象中读取数据（读文件），或者把数据写入这个文件对象（写文件）。

操作形式

mode选项
r 读模式，只能读 文件不存在则报错，
w 写模式，只能写 文件不存在则创建，
以上文件存在则清空在打开

rb 二进制读模式 只能读 文件不存在报错
wb 二进制写模式 只能写 文件不存在则创建，
以上存在则清空再打开
有二进制内容 写入则需要encode(“utf-8”)

rb+ 二进制读模式 可读可写 文件不存在报错
wb+ 二进制写模式 可读可写 文件不存在则创建，
以上存在则清空再打开

a 追加写模式，文件存在则追加写入
a+ 追加读写方式
以上存在则追加写入

默认进入为文件末尾
f.seek(0) 设置读写位置为开头
f.truncate(0) 将文章字节阶段为0

with open(path, "r+") as f:
     old = f.read()
     f.seek(0)
     f.write(data)
     f.write(old)
with open(path,"a") as f:
      f.seek(0)
      old = f.read()
      # 全部删除
      f.truncate(0)
      f.write("---")
      f.write("tags: xxx")
      f.write("---")
      f.close()

字符编码

要读取非UTF-8编码的文本文件，需要给open()函数传入encoding参数，例如，读取GBK编码的文件
比如现在我们要读取一个utf-8的文件，但是我们给定的编码是GBK编码，当然他是报错的

with open('./xx.md','r',encoding='gbk',errors='ignore') as f:
    res = f.read()
    print(res)
    f.close()
Traceback (most recent call last):
  File "file_module.py", line 4, in <module>
    res = f.read()
UnicodeDecodeError: 'gbk' codec can't decode byte 0xa3 in position 167: illegal multibyte sequence

如果我们执意要打开，可以给他加上errors=”ignore”

模式介绍

write模式是写入的模式，
你可以反复调用write()来写入文件，但是务必要调用f.close()来关闭文件。当我们写文件时，操作系统往往不会立刻把数据写入磁盘，而是放到内存缓存起来，空闲的时候再慢慢写入。只有调用close()方法时，操作系统才保证把没有写入的数据全部写入磁盘。忘记调用close()的后果是数据可能只写了一部分到磁盘，剩下的丢失了
另外write在写入的是否如果不管是否存在这个文件，都会阐述

a是追加的模式

w+是可读可写，但是他并不会像A一样去追加内容，但是在实验中我们会发现他根本不能读取内容，比如

import os

with open('./xxx.md','w+',encoding='utf-8') as f:
    f.write("xxxx")
    res = f.read()
    print(res)

在这个里面 先写入xxxx但是无法读到xxx 这是因为在写文件的时候它是有一个类似游标的内容来确定位置的，写完xxxx后，游标在xxxx后面，当然就无法读到了

import os

with open('./xxx.md','w+',encoding='utf-8') as f:
    f.write("xxxx")
    f.seek(0,0)

    # file. seek(offset[, whence])
    # whence：0,1,2三个参数，0表示文件开头，1表示当前位置，2表示文件结尾
    # offset:偏移量，可正可负，正数表示向后移动offset位，负数表示向前移动offset位

    res = f.read()
    print(res)

另外我们还可以使用 tell 来获得当前游标的位置

import os

with open('./xxx.md','w+',encoding='utf-8') as f:
    f.write("xxxx")
    print(f.tell()) # res: 4
    f.seek(0,0) # res 0
    res = f.read()
    print(res)
