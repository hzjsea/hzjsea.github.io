初始化错误

刚安装好的MySQL5.7.12，执行service mysqld start时报错，内容如下：
2017-02-19T07:52:53.807647Z 0 [Warning] TIMESTAMP with implicit DEFAULT value is deprecated. Please use –explicit_defaults_for_timestamp server option (see documentation for more details).
2017-02-19T07:52:53.809065Z 0 [ERROR] –initialize specified but the data directory has files in it. Aborting.
2017-02-19T07:52:53.809095Z 0 [ERROR] Aborting

执行mysql start的时候,出现上面的错误,正常情况下 mysql是被systemd托管的, systemd启动的时候会先去执行mysql中的初始化程序mysqld --initialize ,这个程序会在/var/lib/mysql/data 初始化一系列数据, 然后当你第二次启动的时候,就不会出现这个错误,解决的办法,是先清空data下面的数据,然后再重新启动mysql 但是这样做会清空之前的数据,正常的做法应该是把这个文件夹做个备份,然后重新创个文件夹

cd /var/lib/
mv mysql mysql.back
systemctl resatrt mysqld

mysql插入汉字的问题

创建数据库时使用utf8

CREATE DATABASE `mydb` CHARACTER SET utf8 COLLATE utf8_general_ci;

初始化服务器的密码问题
MySQL服务器初始化（从MySQL 5.7开始）：在服务器初始启动时，如果服务器的数据目录为空，则会发生以下情况：

  • 服务器初始化
  • 在数据目录中生成SSL证书和密钥文件。
  • 该validate_password插件安装并启用。
  • 创建超级用户‘root’@’localhost’,超级用户的密码被设置并存储在错误日志文件中。

查询超级用户的密码

sudo grep

修改密码

-- 先登陆
mysql -uroot -p123456
-- 修改密码
alter user 'root'@'localhost' identified by '123456';
flush all privileges

修改密码出现simple password not pass
ERROR 1820 (HY000): You must reset your password using ALTER USER statement before executing this statement.

解决方法,先修改密码策略

show variables like '%password%';
set global validate_password_policy=0;
set global validate_password_length=4;

-- 关闭mysql

-- 5.6更新的方式

mysql无法插入汉字

查看数据库

show VARIABLES like "character%";

如果数据不重要的话，一劳永逸的解决办法是，使用 alert database tuanplus character set utf8 更改数据库的编码格式，再重新建表。

alert database tuanplus character set utf8; -- 这里tuanplus是数据库

如果项目中的数据是重要数据的话，使用 alter table address convert to character set utf8 转换需要插入汉字的数据表编码为utf8即可（此例中的数据表是address）:

alter table address convert to character set utf8; -- 这里address 是表
set character_set_results=utf8;
set character_set_database=utf8;
set character_set_server=utf8;
set character_set_connection=utf8;
set character_set_client=utf8;

但重启mysql服务后，又还原了，得添加my.ini文件。然后再重启。

错误 找不到sock文件

[root@mysql ~]# mysql
ERROR 2002 (HY000): Can't connect to local MySQL server through socket '/data/mysql/my3306/mysql.sock' (2)

这是是因为my.cnf这个配置文件中指定的sock文件路径中不存在这个文件 这个文件是s类型的好像还看不来。。
配置该路径

[client]
  port=3306
  socket = /var/lib/mysql/mysql.sock

重启mysqld

systemctl restart mysqld

mysql严格模式
mysql严格模式，禁止通过insert的方式直接修改mysql库中的user表进行添加新用户。
报错内容

ERROR 1364 (HY000): Field ‘ssl_cipher’ doesn’t have a default value

修改my.cnf上面的内容

# 原来的
sql-mode=STRICT_TRANS_TABLES,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION
# 修改后的
sql-mode=NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION

# 修改后重启

systemctl restart mysqld

mysql驱动器报错
这个报错内容有两种情况

  • 使用MySql数据库服务为最新版本8.0.11，而在项目开发的过程中使用了低版本的MySql驱动包
  • 当MySQL的数据库隔离级别为REPEATABLE-READ时，查询一个select语句也算是事物的开始，而且在hibernate里会把以select语句开头的事务标记为只读事务，此时在这个事务里再执行insert、update、delete等DML语句就会出现如上的异常信息。

报错2解决：
查看mysql数据库隔离级别

mysql root@10.0.6.55:mysql> SHOW VARIABLES LIKE '%iso%';
+---------------+-----------------+
| Variable_name | Value           |
+---------------+-----------------+
| tx_isolation  | REPEATABLE-READ |
+---------------+-----------------+
1 row in set
Time: 0.024s

查看mysql驱动器版本

mysql root@10.0.6.55:mysql> select version();
+-----------+
| version() |
+-----------+
| 5.6.46    |
+-----------+
1 row in set
Time: 0.031s

修改 REPEATABLE-READ 为READ-COMMITED

Could not retrieve transaction read-only status from server

mysql远程登录授权错误

1045-Access denied for user ‘root’@’’(using password:YES)

这个是因为用户只有在本机登录数据库的权限，没有远程登录的权限，如下

mysql root@10.0.6.55:mysql> select user,host,password from mysql.user;
+-------+-----------+-------------------------------------------+
| user  | host      | password                                  |
+-------+-----------+-------------------------------------------+
| root  | localhost | *6BB4837EB74329105EE4568DDA7DC67ED2CA2AD9 |
| root  | xxx       |                                           |
| root  | 127.0.0.1 |                                           |
| root  | ::1       |                                           |
|       | localhost |                                           |
|       | xxx       |                                           |
| root  | %         | *6BB4837EB74329105EE4568DDA7DC67ED2CA2AD9 |
| hzj   | %         | *05B00DF4BF36116F758117E401D26485E57DB407 |
| hzjqq | localhost | hzjqq12.3                                 |
| hzj   | localhost | *05B00DF4BF36116F758117E401D26485E57DB407 |
| zz    | localhost | *05B00DF4BF36116F758117E401D26485E57DB407 |
+-------+-----------+-------------------------------------------+

这个里面对zz用户名只允许localhost这个IP登录 也就是’127.0.0.1’，如果要开放所有ip，就需要添加一个用户

insert into mysql.user(host,user,password) values("%","zz",password("upyun123"));
# 刷新权限
flush privileges;

vim /etc/my.cnf

validate_password_number_count=3
validate_password_special_char_count=0
validate_password_length=3
