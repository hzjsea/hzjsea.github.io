单容器多应用安装

现在有一个需求,就是需要把一个python的应用程序+mysql数据库,然后安装到容器云上面,但是有一个问题,我们不能开启特权模式
一般情况下mysql默认是使用systemd托管的,也就是我们平常使用的systemctl start mysqld 是如果我们确定不使用docker中的特权模式的话,我们还有另外一种使用方法, 安装Mysqld的二进制包, 二进制包中有一个support-files中会有一个mysql.server启动脚本

dockerfile 如下

FROM centos:7

# 启动命令
# mysqld --initialize --user=root

# 准备工作
RUN yum install -y iproute python-setuptools hostname inotify-tools yum-utils which jq numactl wget repo-release libaio  python3-pip \
# 清除缓存
&&  yum clean all \
&& wget https://dev.mysql.com/get/Downloads/MySQL-5.7/mysql-5.7.28-linux-glibc2.12-x86_64.tar.gz \
&& tar -zxvf mysql-5.7.28-linux-glibc2.12-x86_64.tar.gz \
&& mv mysql-5.7.28-linux-glibc2.12-x86_64 /usr/local/mysql \
&& easy_install supervisor 

COPY supervisord.conf /etc/supervisord.conf
COPY xx.sh .
ADD init.tgz /


#!/bin/bash
echo "export PATH=$PATH:/usr/local/mysql/bin" >> /etc/profile
source /etc/profile
rm -rf mysql-5.7.28-linux-glibc2.12-x86_64.tar.gz

mkdir /usr/local/mysql/log
mkdir /usr/local/mysql/tmp
mkdir /usr/local/mysql/run


# 初始化mysql
mysqld --initialize --user=root

for i in {1..10};do
    sleep 1
done

mysqld_safe --user=root &

本机yum安装

检查是否存在mysql,有则删除

rpm -qa | grep mysql | xargs -I {} rpm -e --nodeps {}
yum clean  all
yum makecache

更新yum 以及 添加yum源，为了下载的速度，我们添加一个阿里源

yum update -y
# 安装wget工具
sudo yum install -y wget 
# 下载源 虽然下的是80的 但可以在里面找到57的源
mkdir /root/package && cd /root/package
wget https://dev.mysql.com/get/mysql80-community-release-el7-3.noarch.rpm
# 本地添加mysql源
sudo yum localinstall mysql80-community-release-el7-3.noarch.rpm -y 
# 安装yum 工具
sudo yum install -y yum-utils -y  
# 查看可用enable的mysql源
yum repolist enabled | grep "mysql.*-community.*"
# 查看所有mysql源
yum repolist all  | grep mysql

关闭8.0镜像,开启5.7镜像
可以看一下这两个包的内容，里面分别配置了一些mysql各个版本存储的地址

yum-config-manager --disable mysql80-community
yum-config-manager --enable mysql57-community
# 查看当前启用的仓库
yum repolist enable | grep mysql

下载安装

yum install mysql-community-server
systemctl start mysqld
systemctl status mysqld

# 查看初始密码
oldpassword=$(sudo grep 'temporary password' /var/log/mysqld.log  | awk -F "localhost: " '{print $2}')
# 重置密码
mysql -u root -p$oldpassword -e "ALTER USER 'root'@'localhost' IDENTIFIED BY 'MyNewPass4!';"

初始化mysql

# 取消密码复杂性设置
cat << EOF >> /etc/my.cnf
validate_password_policy=0
validate_password_mixed_case_count=0
validate_password_number_count=3
validate_password_special_char_count=0
validate_password_length=3
EOF
# 修改完成之后需要重启myql
systemctl restart mysqld

# 重置密码以及远程连接配置
mysql -u root -pMyNewPass4! -e "ALTER USER 'root'@'localhost' IDENTIFIED BY 'upyun123';CREATE USER 'hzj'@'%' IDENTIFIED WITH mysql_native_password BY 'upyun123';GRANT ALL PRIVILEGES ON *.* TO 'hzj'@'%';"

mysql更换国内镜像源的问题

自己的博客那边镜像源用的是国内的，因为当初打的时候，公司有vpn 所以不用考虑速度的问题，后面去了数据中心，需要考虑速度的问题了，就要切换成国内镜像源了
https://mirrors.tuna.tsinghua.edu.cn/help/mysql/
上面这个是清华的镜像源
先删除yum.repos.d下面的镜像包

rm -rf  /etc/yum.repos.d/mysql80-community.repo

创建一个镜像源

cd /etc/yum.repos.d && touch mysql-community.repo
cat <<  EOF > mysql-community.repo
[mysql-connectors-community]
name=MySQL Connectors Community
baseurl=https://mirrors.tuna.tsinghua.edu.cn/mysql/yum/mysql-connectors-community-el7-$basearch/
enabled=1
gpgcheck=1
gpgkey=https://repo.mysql.com/RPM-GPG-KEY-mysql

[mysql-tools-community]
name=MySQL Tools Community
baseurl=https://mirrors.tuna.tsinghua.edu.cn/mysql/yum/mysql-tools-community-el7-$basearch/
enabled=1
gpgcheck=1
gpgkey=https://repo.mysql.com/RPM-GPG-KEY-mysql

[mysql-5.6-community]
name=MySQL 5.6 Community Server
baseurl=https://mirrors.tuna.tsinghua.edu.cn/mysql/yum/mysql-5.6-community-el7-$basearch/
enabled=0
gpgcheck=1
gpgkey=https://repo.mysql.com/RPM-GPG-KEY-mysql

[mysql-5.7-community]
name=MySQL 5.7 Community Server
baseurl=https://mirrors.tuna.tsinghua.edu.cn/mysql/yum/mysql-5.7-community-el7-$basearch/
enabled=1
gpgcheck=1
gpgkey=https://repo.mysql.com/RPM-GPG-KEY-mysql

[mysql-8.0-community]
name=MySQL 8.0 Community Server
baseurl=https://mirrors.tuna.tsinghua.edu.cn/mysql/yum/mysql-8.0-community-el7-$basearch/
enabled=1
gpgcheck=1
gpgkey=https://repo.mysql.com/RPM-GPG-KEY-mysqld
EOF

yum clean all 
yum makecache

mysql无法启动的问题

刚安装好的MySQL5.7.12，执行service mysqld start时报错，内容如下：
2017-02-19T07:52:53.807647Z 0 [Warning] TIMESTAMP with implicit DEFAULT value is deprecated. Please use –explicit_defaults_for_timestamp server option (see documentation for more details).
2017-02-19T07:52:53.809065Z 0 [ERROR] –initialize specified but the data directory has files in it. Aborting.
2017-02-19T07:52:53.809095Z 0 [ERROR] Aborting

这个问题主要是因为数据库本地中已经存在了数据了，需要把他备份删除
查看本地的数据存放点
cat /etc/my.cnf | grep datadir
备份删除数据
mv mysql mysql.back
重启数据库
systemctl restart mysqld

莫名的错误

Filling help tables...2019-12-24 16:46:55 0 [Warning] TIMESTAMP with implicit DEFAULT value is deprecated. Please use --explicit_defaults_for_timestamp server option (see documentation for more details).

解决办法就是在配置文件中添加一段内容
explicit_defaults_for_timestamp=true
