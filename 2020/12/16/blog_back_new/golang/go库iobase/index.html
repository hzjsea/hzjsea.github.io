go库iobase

golang中常常喜欢在一个表示某一动作的结尾加上er
在英语的词缀中 er表示重复的动作
io库定义了一系列的接口,作为底层调用,使之成为共享的公共接口，这些公共接口抽象出了泛用的函数并附加了一些相关的原语的操作。

https://studygolang.com/static/pkgdoc/pkg/io.htm#pkg-variables

  • Reader 读取接口
  • Writer 写入接口
  • Closer 关闭接口
  • Seeker 位移接口
  • WriterAt
  • ReaderAt

type Reader

Read方法读取len(p)字节数据写入p。它返回写入的字节数和遇到的任何错误。即使Read方法返回值n < len(p)，本方法在被调用时仍可能使用p的全部长度作为暂存空间。如果有部分可用数据，但不够len(p)字节，Read按惯例会返回可以读取到的数据，而不是等待更多数据。
也就是说 假设你有一个12字节的byte数组 去读取一个16字节的字符串,返回的n是12个字节

type Writer

Write方法len(p)字节数据从p写入底层的数据流。它会返回写入的字节数(0 <= n <= len(p))和遇到的任何导致写入提取结束的错误。Write必须返回非nil的错误，如果它返回的n < len(p).Write不能修改切片p中的数据，即使临时修改也不行

type Closer

Closer接口用于包装基本的关闭方法。
